# TDD Enforcement Skill

## Purpose

Enforces Test-Driven Development methodology throughout the development workflow, ensuring tests are written before implementation code.

## When to Use

- When implementing new features (tests first)
- When fixing bugs (reproduction test first)
- When refactoring code (preserve test coverage)
- When establishing development standards

## Key Features

- TDD cycle enforcement (Red-Green-Refactor)
- Test coverage requirements
- Test-first workflow patterns
- Quality gate integration
